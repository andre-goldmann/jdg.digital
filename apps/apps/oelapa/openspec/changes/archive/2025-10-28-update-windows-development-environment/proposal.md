# Update Windows Development Environment Documentation

## Why
The project documentation needs to reflect the actual development environment and tools used by the team. Currently, the documentation lacks specific guidance for Windows developers and doesn't mention the testing framework or development assistance tools being used.

## What Changes
- Add Windows 10/11 as the specified development environment
- Document PowerShell as the primary command-line interface with correct syntax examples
- Specify Playwright as the end-to-end testing framework
- Include Angular MCP and Playwright MCP as development assistance tools
- Update command examples to use PowerShell syntax (`;` instead of `&&`)

## Impact
- Affected specs: project-documentation (new capability)
- Affected code: openspec/project.md documentation file
- Benefits: Clearer onboarding for new developers, accurate development environment setup
